package.json文件

2017-04-08

现在的前端项目根目录下都会有一个package.json里面定义了项目的名称、版本、依赖等，npm install根据这个文件自动配置依赖。

下面一个实际的package.json

  • name 项目的名称，必须
  • version 项目版本 A.B.C（大版本.次要版本.小版本）

这里顺便对这个 A.B.C版本号格式的做一下说明

A 表示大版本号，一般当软件整体重写，或出现不向后兼容的改变时，增加A，A为零时表示软件还在开发阶段。B 表示功能更新，出现新功能时增加BC 表示小修改，如修复bug，只要有修改就增加C除了版本号之外还会有一些修饰的词，比如：alpha: 内部版本beta: 测试版rc: 即将作为正式版发布lts: 长期维护

script

scripts指定了运行脚本命令的npm命令行缩写，比如start指定了运行npm run start时，所要执行的命令。

上面贴出的package.json文件就分别定义了 npm run dev, npm run build, npm run lint文件。

dependencies字段，devDependencies字段

前者指定项目生产环境的依赖模块，后者指定开发环境的依赖模块。

指定版本的规则如下

  • 指定版本：比如1.2.2，遵循“大版本.次要版本.小版本”的格式规定，安装时只安装指定版本。
  • 波浪号（tilde）+指定版本：比如~1.2.2，表示安装1.2.x的最新版本（不低于1.2.2），但是不安装1.3.x，也就是说安装时不改变大版本号和次要版本号。
  • 插入号（caret）+指定版本：比如ˆ1.2.2，表示安装1.x.x的最新版本（不低于1.2.2），但是不安装2.x.x，也就是说安装时不改变大版本号。需要注意的是，如果大版本号为0，则插入号的行为与波浪号相同，这是因为此时处于开发阶段，即使是次要版本号变动，也可能带来程序的不兼容。
  • latest：安装最新版本。
