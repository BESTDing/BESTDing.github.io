类的继承

JavaScript中的继承是通过原型链实现的，假如有一个类的原型是另一个类的实例，那么这个类的实例有一个__proto__属性指向原型就是父类的实例，而这个原型也有一个__proto__属性指向自己的原型。这样就形成原型链，画个图更好理解点。

原型继承

以上就是原型链继承，原型链继承有两个问题

  1. 如果原型中包含引用类型的属性，所有类实例共享这个属性，一个实例修改会影响其他实例。

借用构造函数继承

借用构造函数解决了原型链继承问题，但是有么有发现我们并没有调用b.print(),我们只是继承了属性并没有继承方法，方法定义在父类的原型上。

组合继承

顾名思义组合继承就是结合借用构造函数和原型链继承两种方法的优点的一种类的继承方法。

这样我们即继承了属性又继承了方法，其实方法也是属性，这样说为了区分。

说到这里我们看一下es6的继承

我们看子类调用了父类的构造函数是不是和上面借用构造函数相似,其实本质上es6的继承就是es5的组合继承，class和extend两个关键字只是语法糖

在es6中有两条继承链，一是构造函数的继承，二是原型的继承

子类的__proto__总是指向父类即B.__proto__ === A, 原型继承即方法的继承， B.prototype.__proto__ === A.prototype就是我们第一步讲的原型链。
方法的继承没什么卵用，实际上我们还要通过super()手动调用父类的构造方法。所以es6 class和extend实质上就是es5的组合继承。

有一点不同

es5中我们以这种形式调用父类的构造函数A.call(this, x)可见我们先创建子类实例然后复制父类属性。但是es6 中我们先调用super()创建父类的实例(其实也不是什么父类的实例了自我理解就是一个空对象)然后在添加子类的自定义属性。现在假设有如下继承链A->B->C->D 我们现在想创建一个D的实例，那我们调用D的构造函数，D的构造函数又会调用C的构造函数，以此类推最先在A上创建this对象，然后添加属性，可见es6的继承是自上而下的，最顶层的祖先的属性最先被添加。(其实es5组合继承也是自顶向下的这样才符合常理嘛唯一的不同就是创建this的先后了)

JavaScript正则表达式总结

2017-07-05

正则表达式基础

正则表达式中的转义字符

  • \o NUL字符(\u0000)
  • \t 制表符(\u0009)
  • \n 换行符(\u000A)
  • \v 垂直制表符(\u000B)
  • \f 换页符(\u000C)
  • \r 回车符(\u000D)

另外以下字符在正则表达式中有特殊含义，想要匹配这些字符有的要加上前缀\

^ $ . * + ? = ! : | \ / ( ) [ ] { }

字符类

  • [....] 匹配方括号内的任意字符
  • [^...] 不包括在方括号内的任意字符
  • . 除换行符和其他Unicode行中支付之外的任意字符
  • \w 任何ASCII字符组成的单词
  • \s 任何Unicode空白符
  • \W 任何不适ASCII字符组成的单词
  • \S 任何非Unicode空白符的字符
  • \d 任何ASCII数字
  • \D 除了ASCII数字之外的任何字符
  • [\b] 退格直接量

重复匹配

  • {n, m} 匹配一项至少n次，但不能超过m次
  • {n, } 匹配前一项n次或者更多次
  • {n} 匹配前一项n次
  • ? 匹配前一项0次或者1次
  • + 匹配一次货多次
  • * 匹配0次或多次

选择，分组，引用

  • |匹配该符号左边或者右边的表达式，匹配顺序从左到右，如果左边的表达式先匹配成功就不会再匹配右边的表达式。
  • () 组合将几个单项组合为一项，我们可以在匹配结果中引用各个分组的匹配内容。
  • (?:) 只是组合，并不生成引用，也不记忆与该组相匹配的字符
  • \n 引用第n个分组匹配的结果，n的计数是从左到右的左括号数， (?: 不计数

指定位置进行匹配

  • ^ 匹配字符串的开头，在多行检索中，匹配一行的开头
  • $ 匹配字符串的结尾
  • \b 匹配一个单词的边界
  • \B 匹配非单词的边界位置
  • (?=p) 零宽正向先行断言，要求接下来的字符都与p匹配，但不能包括匹配p的那些字符
  • (?!p) 零宽负向先行断言，要求接下来的字符不与p匹配

修饰符

  • i 执行不区分大小写的匹配
  • g 执行一个全局匹配，找到所有匹配，而不是匹配一个后停止匹配
  • 多行匹配模式， ^匹配一行的开头和字符串的开头，$匹配行的结束和字符串的结束

String方法

string方法是在字符串上调用的支持正则表达式的方法总共有4个

search()

search()的参数是一个正则表达式，如果不是正则表达式，会调用RegExp()构造函数转化为正则表达式。不支持全局搜索，忽略修饰符g
search()返回第一个匹配子串的位置，没有匹配返回-1,

replace()
replace()用于检索替换操作，第一个参数可以是正则表达式，也可以是字符串，如果是字符串会直接在目标字符串内搜索这个子串并替换第二个参数的字符串。另外如果设置了g修饰符，目标字符串内所有匹配的子串都将被替换。

match()

match()唯一一个参数是一个正则表达式，如果不是正则表达式，首先转换为正则表达式。
如果设置了全局修饰符gmatch()返回一个数组包含所有匹配结果，如果没有设置g修饰符，match()返回一个数组，第一个元素是第一个匹配的结果，其他元素是分组匹配的结果。

split()

用于将字符串拆分为一个子串组成的数组，分隔符参数可以为字符串也可以为正则表达式。

RegExp对象

  • 构造函数 RegExp()接受两个参数，第一个参数是正则表达式两条斜线之间的文本 对于/a_/ 就是 'a_',第二个参数是可选的，修饰符组合
  • RegExp对象的属性
    • source 只读字符串包含正则表达式文本
    • global 只读布尔值， 说明是否带有g全局修饰符
    • ignoreCase 只读布尔值， 说明是否带有i修饰符
    • multiline 只读布尔值，是否带有 m修饰符
    • lastIndex 可读可写的整数， 存储在整个字符串中下一次检索的位置。
  • RegExp对象的方法
    • exec()的参数是一个字符串，返回结果是一个数组，第一个元素是匹配结果，之后元素是分组匹配的的结果，index属性是匹配的位置，input属性是正在检索的字符串。
    • test() 返回结果是一个布尔值，当exec()
      的返回结果不是null时，返回true.

ES6新特性

  • 构造函数， ES6构造函数可以接受正则表达式作为参数，返回原有正则表达式的一个拷贝。另外如果修改了修饰符，则忽略原有表达式的修饰符。
  • u 修饰符
  • y 修饰符 和g相似，不同的是，要求下次匹配位置必须和lastIndex相同。
  • sticky属性，表示是否设置了y修饰符
  • flags属性， 返回正则表达式的修饰符

RegExp.escape()

如果字符串包含了正则表达式的一些有特殊意义的字符，则字符串必须经过转义才能传入构造函数，创建正则表达式对象。

现在的前端项目根目录下都会有一个package.json里面定义了项目的名称、版本、依赖等，npm install根据这个文件自动配置依赖。

下面一个实际的package.json

  • name 项目的名称，必须
  • version 项目版本 A.B.C（大版本.次要版本.小版本）

这里顺便对这个 A.B.C版本号格式的做一下说明

A 表示大版本号，一般当软件整体重写，或出现不向后兼容的改变时，增加A，A为零时表示软件还在开发阶段。B 表示功能更新，出现新功能时增加BC 表示小修改，如修复bug，只要有修改就增加C除了版本号之外还会有一些修饰的词，比如：alpha: 内部版本beta: 测试版rc: 即将作为正式版发布lts: 长期维护

script

scripts指定了运行脚本命令的npm命令行缩写，比如start指定了运行npm run start时，所要执行的命令。

上面贴出的package.json文件就分别定义了 npm run dev, npm run build, npm run lint文件。

dependencies字段，devDependencies字段

前者指定项目生产环境的依赖模块，后者指定开发环境的依赖模块。

格式大致如下

指定版本的规则如下

  • 指定版本：比如1.2.2，遵循“大版本.次要版本.小版本”的格式规定，安装时只安装指定版本。
  • 波浪号（tilde）+指定版本：比如~1.2.2，表示安装1.2.x的最新版本（不低于1.2.2），但是不安装1.3.x，也就是说安装时不改变大版本号和次要版本号。
  • 插入号（caret）+指定版本：比如ˆ1.2.2，表示安装1.x.x的最新版本（不低于1.2.2），但是不安装2.x.x，也就是说安装时不改变大版本号。需要注意的是，如果大版本号为0，则插入号的行为与波浪号相同，这是因为此时处于开发阶段，即使是次要版本号变动，也可能带来程序的不兼容。
  • latest：安装最新版本。
